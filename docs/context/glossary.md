# Glossary

Термінологія проєкту. Використовується агентами для однозначного розуміння понять.

---

## Доменні терміни

**Project**
Контейнер верхнього рівня для перекладів. Відповідає одному продукту або середовищу. Ідентифікується `slug`-ом у URL.

**Slug**
Унікальний людиночитабельний ідентифікатор проєкту. Використовується замість UUID у всіх URL. Формат: `^[a-z0-9]+(?:-[a-z0-9]+)*$`.

**Language**
Мова, підключена до проєкту. Зберігає ISO-код (`en`, `uk`, `zh-CN`). Один проєкт — кілька мов.

**Base Language (Source Language)**
Еталонна мова проєкту. Перша додана мова стає базовою автоматично. Може бути змінена вручну. Не може бути видалена.

**Translation Key (Key)**
Ідентифікатор текстового рядка у dot-notation. Мінімум 2 сегменти: `auth.login.title`. Зберігає опис (контекст) для перекладачів.

**Dot-notation**
Формат запису ключів через крапку: `feature.component.label`. Визначає ієрархію при nested-експорті.

**Nesting Conflict**
Ситуація, коли ключ і його prefix-сегмент існують одночасно: наявність `menu` блокує створення `menu.file` і навпаки. Backend повертає 400.

**Translation Value (Translation)**
Значення ключа для конкретної мови. Порожній рядок `""` або відсутнє значення означає, що переклад відсутній.

**Untranslated**
Статус ключа для конкретної мови: значення відсутнє або порожній рядок. Derived state — не зберігається в БД явно.

**Translated**
Статус ключа для конкретної мови: значення непорожнє.

## UI-терміни

**Inline Editing**
Редагування значення перекладу безпосередньо у рядку таблиці ключів. Без переходу на інший екран.

**Side Panel**
Бокова панель, що відкривається для розширеного редагування ключа. Показує всі мовні варіанти та опис ключа.

**Bulk Delete**
Масове видалення кількох обраних ключів за одну операцію.

**Export (flat)**
Експорт перекладів у плаский JSON: `{ "auth.login.title": "Sign In" }`.

**Export (nested)**
Експорт перекладів у вкладений JSON: `{ "auth": { "login": { "title": "Sign In" } } }`.

## Технічні терміни

**Access Token**
JWT-токен для авторизації запитів. TTL — 60 хв. Передається у заголовку `Authorization: Bearer <token>`.

**Refresh Token**
Токен для оновлення access token. TTL — 7 днів. Зберігається у httpOnly cookie. Недоступний через JS.

**Token Rotation**
Після кожного `/token/refresh/` видається нова пара токенів. Старий refresh token одразу інвалідується.

**RTL (Right-to-Left)**
Напрямок тексту справа наліво. Застосовується для мов: `ar`, `he`, `fa`, `ur`. На input-ах встановлюється `dir="rtl"`.

**LimitOffset Pagination**
Тип пагінації: параметри `limit` (розмір сторінки, default 20, max 100) та `offset` (зміщення). `count` — загальна кількість записів.

**FSD (Feature-Sliced Design)**
Архітектурна методологія організації коду. Шари: `app → pages → widgets → features → entities → shared`. Верхні шари імпортують з нижніх, не навпаки.

**Slice**
Один модуль всередині шару FSD. Наприклад, `entities/project` або `features/create-project`. Експортує публічний API через `index.ts`.
